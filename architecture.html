Tower Defense — How It's Built

A map for the team. No prior code experience needed to read this.

🔵 Person A — Game Logic

Your file: server/GameRoom.js

All the numbers that control how the game feels are constants at the very top of this file. Change them and save — the server restarts automatically.

Things to tweak: enemy speed, HP, gold rewards, tower damage, tower range, wave sizes, lives.

🟢 Person B — Visuals

Your file: client/src/GameScene.js

Everything you see on screen is drawn here. Enemies are red rectangles, towers are blue rectangles — change colors, sizes, or swap in images.

The path is drawn using the WAYPOINTS array at the top. Keep it in sync with GameRoom.js if you change it.

🟣 Person C — UI & Buttons

Your file: client/src/UIScene.js

The HUD bar at the top, the Place Tower button, the Start Wave button — all here. Add new buttons, score displays, wave countdowns, or a win screen.

This scene runs on top of the game canvas and never interferes with gameplay.

How to start each session: Open VS Code → open a terminal → git pull to get the latest from your teammates → start the server (cd server && npm run dev) and client (cd client && npm run dev) in two terminals → open http://localhost:5173 in your browser → start building.

When something works, commit it: git add . && git commit -m "what I changed" && git push
